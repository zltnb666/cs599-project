# Product Spec - DocuMind 智能文档问答系统

## 1. 项目来源

本项目基于开源项目 [ask-multiple-pdfs](https://github.com/alejandro-ao/ask-multiple-pdfs) 进行改造。

**原项目简介：**
- 功能：上传多个 PDF 文件，基于内容进行问答
- 技术栈：Streamlit + LangChain + OpenAI
- 优点：代码简洁，功能完整
- 不足：仅支持 OpenAI API，成本高；缺少引用来源；无对话历史管理

## 2. 改造目标

针对企业文档问答场景，进行以下改造：

### 2.1 降低使用成本
- **问题**：OpenAI API 成本高
- **方案**：替换为 DeepSeek API，成本降低 

### 2.2 提升答案可信度
- **问题**：回答无法溯源，用户不知道答案来自哪里
- **方案**：添加引用来源展示，标注答案出处

### 2.3 改善用户体验
- **问题**：无对话历史管理，无法查看之前的问答
- **方案**：添加对话历史导出功能

### 2.4 增加数据统计
- **问题**：无法了解系统使用情况
- **方案**：添加文档统计、对话统计面板

## 3. 核心功能

### 3.1 原有功能（保留）
- ✅ 上传多个 PDF 文件
- ✅ 文档内容向量化存储
- ✅ 基于文档内容的智能问答
- ✅ 多轮对话支持

### 3.2 新增功能（改造）
- 🆕 支持 DeepSeek API
- 🆕 引用来源展示
- 🆕 对话历史导出
- 🆕 文档和对话统计面板
- 🆕 错误处理优化

## 4. 用户故事

**作为企业员工：**
- 我希望上传公司的产品手册、技术文档后，能快速找到答案
- 我希望看到答案的来源，确认信息的准确性
- 我希望导出对话记录，方便后续查阅

**作为企业管理员：**
- 我希望使用成本更低的 API，降低运营成本
- 我希望了解系统使用情况，优化文档管理

## 5. 技术选型

**LLM 模型：**
- 原始项目：OpenAI GPT-3.5
- 改造后：DeepSeek Chat
- 改进点：成本降低

**向量数据库：**
- 原始项目：FAISS
- 改造后：FAISS（保持）
- 改进点：无需改动

**UI 框架：**
- 原始项目：Streamlit
- 改造后：Streamlit（保持）
- 改进点：无需改动

**向量化模型：**
- 原始项目：OpenAI Embeddings
- 改造后：OpenAI Embeddings
- 改进点：保持兼容性

## 6. 项目价值

### 6.1 用户价值
- 提升答案可信度（引用来源）
- 改善使用体验（历史导出）
- 增强数据透明度（统计面板）

### 6.2 技术价值
- 学习开源项目改造方法
- 掌握 LLM API 替换技巧
- 理解企业级应用需求
